# Community Promotion Plan

> 更新日期：2026-03-05。目标：把 `lucy-agent-mesh` 推进到 Agent 社区、区块链社区、以及 AI x Blockchain 交叉社区。

## 核心定位（对外统一口径）
一句话：**Agent 之间的微信底座（P2P、本地优先、MCP-first、可绑定链上身份）**。

只讲三件事：
1. 不依赖中心消息服务器，节点可断网恢复。
2. Agent 可直接通过 MCP 工具接入，不需要先写一层中间后端。
3. 链上承载身份与权限锚点，通信正文默认链下。

## 渠道优先级（先小后大）
### P0：立即投入（高匹配 + 可快速出案例）
- MCP Registry 与相关社区：提交 server 信息与接入教程，争取被 Agent 工具开发者直接试用。  
  入口：<https://modelcontextprotocol.io/introduction> / <https://registry.modelcontextprotocol.io/>
- Solana 开发者入口：围绕“链上身份绑定 + 链下通信”写实战文。  
  入口：<https://solana.com/developers> / <https://solana.stackexchange.com/>
- Superteam：发布 bounty 风格任务，收集真实集成案例。  
  入口：<https://www.superteam.fun/>

### P1：1-2 个月推进（扩大触达）
- ElizaOS 社区：发布“如何把 agent runtime 接到 P2P mesh”的适配示例。  
  入口：<https://docs.elizaos.ai/>
- Colosseum（Solana 黑客生态）：围绕 Agent + Solana 场景做 demo 提案与报名。  
  入口：<https://www.colosseum.org/>
- Solana Foundation 社区渠道（论坛/Discord）定期同步版本更新。  
  入口：<https://solana.org/>

## 内容与活动节奏（30-60-90 天）
1. 0-30 天：发布 3 篇硬核教程（本地双节点、MCP 接入、Solana 身份绑定），每篇附可复现实验步骤。
2. 31-60 天：发起 1 次线上 workshop，主题固定为“30 分钟把两个 agent 拉进同一 mesh 并完成私聊/群聊”。
3. 61-90 天：启动 community challenge（例如“做一个团队协作 agent mesh”），产出案例库与排行榜。

## 指标（只追可验证指标）
- 集成指标：新增 MCP 客户端接入数、被引用到第三方 Agent 项目的次数。
- 社区指标：教程复现成功反馈数、社区问答响应时长、活跃贡献者数。
- 产品指标：双节点连通成功率、消息送达成功率、dead-letter 下降趋势。

## 执行原则
1. 先案例后口号：每次对外发布都附可运行命令或截图证据。
2. 先开发者后大众：当前阶段不做“非技术用户”叙事，聚焦工程团队可落地场景。
3. 先交叉社区再泛社区：优先 AI x Blockchain 的开发者群体，转化效率更高。
